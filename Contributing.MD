# Contributing to FPCOS

Thank you for your interest in contributing to First Principle Codex OS.

---

## What We Accept

### ✅ Welcome Contributions
- Corrections to factual errors in reference files
- Additional cross-domain pattern examples in `references/COMPOUND_MIND_PATTERNS.md`
- New example skill implementations (finance, medical, legal, engineering, etc.)
- Translation of documentation to other languages
- Bug reports on logical inconsistencies in the layer system

### ❌ Not Accepted
- Changes to the 5 core Skill Interface Contract requirements (L0, L1, L4, L5 are fixed)
- Modifications that weaken the Kalama10 proof standard
- Removal of the Shadow Gate or any of its 5 protocols
- License term modifications
- Changes that reduce attribution requirements

---

## How to Contribute

### Reporting Issues
Use GitHub Issues with the appropriate template:
- **Bug report**: logical inconsistency, unclear protocol, broken example
- **Skill proposal**: new example skill that inherits FPCOS

### Pull Requests

1. Fork the repository
2. Create a branch: `feature/your-description` or `fix/your-description`
3. Make your changes
4. Ensure all example skills still comply with the Skill Interface Contract
5. Submit PR with a clear description

### PR Checklist
- [ ] Attribution line preserved in all modified files
- [ ] No changes to L0, L1, L4, or L5 core contracts
- [ ] No changes to Kalama10 10-rejection standard
- [ ] License terms unchanged
- [ ] New example skills include `base_os` and `base_os_version` in frontmatter

---

## Code of Conduct

Contributions must be made in good faith, with the goal of improving
accuracy, clarity, and usefulness of the System. Contributions that
introduce deliberate ambiguity or weaken verification standards will be rejected.
