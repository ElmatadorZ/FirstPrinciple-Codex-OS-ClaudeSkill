## Cross-Domain Pattern Library

### Pattern 1 — Feedback Loop Universality
Origin: Control systems
Appears in: Markets, biology, software, coffee roasting, organizations
Structure: Output → Sensor → Comparator → Actuator → Output
Insight: Sensor + comparator = leverage point.

### Pattern 2 — S-Curve Adoption
Origin: Diffusion of innovations
Appears in: Tech adoption, startups, skill acquisition, epidemics
Structure: Slow → exponential → saturation
Insight: Position on curve changes optimal strategy.

### Pattern 3 — Constraint Theory
Origin: Manufacturing (Goldratt)
Appears in: Software, supply chains, learning, teams
Structure: System output = weakest link. Non-bottleneck improvement = zero gain.
Insight: Find constraint. Fix only constraint. Repeat.

### Pattern 4 — Information Asymmetry
Origin: Economics (Akerlof)
Appears in: Markets, hiring, negotiations, medical diagnosis
Structure: One party has more information. Less-informed misprices risk.
Insight: Map who has what. Less-informed = exposed.

### Pattern 5 — Phase Transition
Origin: Thermodynamics
Appears in: Market regime changes, culture shifts, revolutions
Structure: Linear behavior → threshold → discontinuous jump
Insight: Linear extrapolation fails near threshold.

### Pattern 6 — Compounding Returns
Origin: Finance
Appears in: Skill, relationships, brand, code quality, reputation
Structure: Return N applied to larger base than N-1
Insight: Small early advantages compound. Time = most undervalued variable.

### Pattern 7 — Pareto Distribution
Origin: Pareto / economics
Appears in: Wealth, code bugs, customer revenue, social reach
Structure: ~20% inputs → ~80% outputs
Insight: Finding the 20% > optimizing the 80%.

### Pattern 8 — Regression to Mean
Origin: Statistics (Galton)
Appears in: Athletic performance, investment returns, exam scores
Structure: Extreme observations → less extreme, independent of intervention
Insight: Many "successful interventions" are regression in disguise.

## Anti-Loop Protocols

Axis Rotation: "I am on [X] axis. What does [Y] axis solution look like?"
Domain Transplant: "How does [different domain] solve an analogous problem?"
First Principle Reset: "What is the actual goal? Am I solving the right problem?"
Inversion Path: "What would make it worse? Now reverse that."

## Token Efficiency Rule

Map ALL paths internally. Output ONLY paths that matter.
Test: "Would knowing this alternative change the decision? If no → omit."
